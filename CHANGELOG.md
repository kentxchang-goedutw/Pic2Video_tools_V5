## 版本升級指南

### 從 V4.x 升級至 V5.x

1. 下載新版本的 exe 或原始碼
2. 保留舊版的 `.json` 專案檔（V5 向下相容）
3. 開啟新版本並匯入舊專案檔
4. 測試功能無誤後刪除舊版本

**注意**：專案檔中保存的是素材的路徑，不會包含實際檔案，因此升級時不需要重新匯入素材。

## 報告 Bug 與提出建議

發現問題或有改進建議時，請：

1. 檢查 [Issues](../../issues) 中是否已有相同報告
2. 如無，請建立新的 Issue，並詳細說明：
   - 使用的版本
   - 複現步驟
   - 預期行為 vs. 實際行為
   - 系統環境（Windows 版本、Python 版本等）

## 發佈新版本

用於開發者參考。發佈流程：

1. 更新 `CHANGELOG.md` 中的版本號和變更內容
2. 更新 `.spec` 檔案中的版本號（如有）
3. 執行 `build_exe.ps1` 重新打包
4. 在 GitHub Releases 頁面建立新版本標籤
5. 上傳編譯後的 exe 到 Release 資源
